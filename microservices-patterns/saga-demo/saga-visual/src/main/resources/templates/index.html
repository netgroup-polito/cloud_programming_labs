Saga Pattern — Visual Demo

Choreography-based saga: distributed transactions without 2PC

LIVE SERVICES — connected to real microservices via REST
This demo shows the Choreography-based Saga pattern. The Order Service creates an order in PENDING state and publishes an event. The Customer Service checks credit and publishes the result. The order is then APPROVED or REJECTED — no distributed transaction needed! Press Auto Demo for a guided walkthrough.

5
Order Service: update order to APPROVED / REJECTED
